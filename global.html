checkRedirect(response)

Checks whether "Location" header is preset. Location header means that this is a redirect response with empty body, so the Response object with simple JSON { redirect_url: 'URL'} will be returned. Otherwise initial Response object is returned.
Parameters:
Name Type Description
response Response Response object
Returns:
Response object either original or with redirect URL
Type
Response

checkStatus(response)

Checks the status code of the Response. If the status code is <= 400 the Response object is returned. Otherwise the Error is thrown.
Parameters:
Name Type Description
response Response Response object
Throws:
- error object accoding to the bad response.
Type
Error
Returns:
Response object
Type
Response

createModel(schema)

Utility function for creating models from schema and source provided. We use convention over configuration approach to define the schema. This function also recognizes complex sources, e.g. arrays of custom objects. Using a simple type validation check the read-only properties are defined for created models.
Parameters:
Name Type Description
schema Object name of attribute.
Returns:
Model - constructor function for a model.
Type
function | Model
